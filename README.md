# Hospital Inpatient Management System (HIMS) 🏥

**HIMS** — это комплексная информационная система для управления стационарным лечением в медицинских учреждениях. Проект построен на современной клиент-серверной архитектуре, объединяющей высокопроизводительный API и интуитивно понятный графический интерфейс.

## 🌟 Основные возможности

* **Управление пациентами:** Полный цикл регистрации, госпитализации и ведения медицинских карт.
* **Управление персоналом:** Иерархия должностей, учет квалификации и графиков работы.
* **Инфраструктура:** Моделирование структуры больницы (отделения, палаты, помещения).
* **Финансы:** Модуль учета платежей и методов оплаты (наличные, карта, страховка).
* **Безопасность:** Четырехуровневая система ролей (от 0 до 3) с динамическим ограничением прав в интерфейсе.
* **Администрирование:** Инструменты для создания и восстановления бэкапов базы данных PostgreSQL через API.
* **Качество кода:** Покрытие системы автоматизированными тестами (pytest) для всех основных роутеров.

## 🛠 Стек технологий

### Backend

* **FastAPI**: Высокопроизводительный веб-фреймворк для создания API.
* **SQLAlchemy (ORM)**: Взаимодействие с базой данных на уровне объектов Python.
* **PostgreSQL**: Профессиональная реляционная СУБД.
* **Pydantic**: Валидация данных и строгая типизация.
* **Jose / Passlib**: Безопасная работа с JWT-токенами и хеширование паролей.

### Frontend

* **PySide6 (Qt for Python)**: Кроссплатформенный GUI-фреймворк для создания десктопных приложений.
* **Requests**: Взаимодействие с API-клиентом.

## 🔐 Уровни доступа

Система реализует ролевую модель:

* **Уровень 0:** Персонал без прямого доступа к системе.
* **Уровень 1 (Сотрудник):** Только просмотр таблиц ввода (пациенты, госпитализации).
* **Уровень 2 (Врач):** Просмотр справочников, добавление и редактирование записей о лечении.
* **Уровень 3 (Администратор):** Полный доступ, управление персоналом, финансами и системными бэкапами.

## 📁 Структура проекта

```text
├── backend/            # Исходный код API (FastAPI)
│   ├── admin/          # Скрипты администрирования и бэкапов
│   ├── routers/        # Эндпоинты по разделам (пациенты, персонал и т.д.)
│   ├── models.py       # Описание таблиц БД (SQLAlchemy)
│   └── schemas.py      # Схемы валидации данных (Pydantic)
├── frontend/           # Исходный код GUI (PySide6)
│   ├── main_gui.py     # Логика главного окна
│   └── forms/          # UI файлы (.ui) и диалоги
├── test-server/        # Тесты (pytest)
└── main.py             # Точка входа для запуска

```

## 🚀 Быстрый старт

### 1. Требования

* Python 3.12+
* PostgreSQL

### 2. Установка

```bash
git clone https://github.com/I-Need-Help-Pls-T-T/HIMS.git
cd HIMS
python -m venv venv
source venv/bin/activate  # Или venv\Scripts\activate на Windows
pip install -r requirements.txt

```

### 3. Настройка окружения

Создайте файл `.env` в корневом каталоге:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=hospital_db
DB_USER=postgres
DB_PASSWORD=your_password
SECRET_KEY=your_secret_string

```

### 4. Запуск

**Запуск сервера:**

```bash
uvicorn backend.main:app --reload

```

**Запуск клиента:**

```bash
python main.py

```

## 🧪 Тестирование

Для запуска тестов и проверки работоспособности всех модулей выполните:

```bash
pytest test-server/

```
